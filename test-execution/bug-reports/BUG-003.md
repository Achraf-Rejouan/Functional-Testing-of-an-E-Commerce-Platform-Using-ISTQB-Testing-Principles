## Description

On the Shopping Cart page, when a user changes the quantity of an existing cart item to **0** (either by:
- Clicking the **minus (−)** button when the quantity is already **1**, or
- Typing **0** directly into the quantity input field)

...the system **immediately and irreversibly removes the item from the cart** without displaying any confirmation prompt, warning dialog, or undo option.

This behavior is a significant UX defect. A user who accidentally clicks the minus button once too many times or accidentally types 0 will lose their cart item with no way to recover it from the cart page — they would need to navigate back to the product page and re-add it from scratch.

Standard e-commerce best practices require either:
1. A minimum quantity guard that prevents the value from dropping below 1 (quantity floor), or
2. A confirmation modal asking the user _"Are you sure you want to remove this item?"_ before proceeding

---

## Steps to Reproduce

1. Navigate to `https://demo.prestashop.com/#/en/front`
2. Add any product to the cart (e.g., Hummingbird Printed T-Shirt, Size M, Color White, Qty 1)
3. Click the **cart icon** in the top navigation → proceed to the cart page
4. Locate the quantity input for the cart item (currently showing **1**)
5. **Click the minus (−) button** once  
   *— OR —*  
   Clear the quantity field and type **0**
6. Press Tab or click elsewhere to confirm

**Expected frequency:** Reproducible 100% of the time.

---

## Expected Result

**Scenario A (Minimum quantity guard):**  
The quantity cannot be set below 1. The minus button is disabled when quantity is 1, or the value snaps back to 1 if 0 is entered. No item is removed.

**Scenario B (Confirmation dialog):**  
A dialog appears asking the user to confirm removal:  
> _"Are you sure you want to remove 'Hummingbird Printed T-Shirt' from your cart?"_  
> [Remove Item] [Cancel]

The item is only removed after the user explicitly confirms.

---

## Actual Result

The cart item is **immediately removed** from the cart the moment the quantity reaches 0. No confirmation dialog is shown. The page auto-refreshes to display:
> _"There are no more items in your cart."_

The action is **irreversible** from the cart page — the only recovery path is to navigate back to the product page and re-add the item.

---

## Impact

- **Business Impact:** Users may accidentally empty their cart, leading to abandoned purchases and lost revenue.
- **UX Impact:** Destroys user trust; high frustration potential, especially on mobile where buttons are small.
- **Reproducibility:** 100% reproducible.

## Suggested Fix

Implement one or both of the following:

**Fix 1 — Quantity floor guard:**
```javascript
quantityInput.addEventListener('change', () => {
  if (quantityInput.value < 1) {
    quantityInput.value = 1; // Snap back to minimum
  }
});
```

**Fix 2 — Confirmation before removal:**  
When quantity reaches 0 (via minus button or direct input), intercept the change and show a modal:  
> _"Setting the quantity to 0 will remove this item from your cart. Do you wish to continue?"_  
> [Yes, Remove] [Keep Item]
