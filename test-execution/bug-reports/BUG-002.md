## Description

When a user navigates to a product detail page (PDP) that has **configurable attributes** (e.g., Size and Color) and clicks **"ADD TO CART"** without explicitly interacting with any attribute selector, the system silently adds the product to the cart using the **pre-selected default values** (Size: S, Color: White).

No warning, tooltip, confirmation modal, or prompt is shown to inform the user that their cart addition was made with default — rather than intentionally chosen — attributes. This leads to a poor user experience and a high likelihood of incorrect orders, as users may not notice the defaults differ from their intended selection.

This is particularly problematic for first-time visitors unfamiliar with the default pre-selection behavior.

---

## Steps to Reproduce

1. Navigate to `https://demo.prestashop.com/#/en/front`
2. Go to **Clothes → Men → Hummingbird Printed T-Shirt**
3. On the Product Detail Page, do **not** interact with the Size dropdown or Color swatches (leave them at their default state: Size S, Color White)
4. Click **"ADD TO CART"** directly

**Expected frequency:** Reproducible 100% of the time.

---

## Expected Result

One of the following behaviors should occur:
- **Option A:** The system displays a prompt or tooltip instructing the user to explicitly select a size and color before adding to cart (e.g., _"Please select your size and color"_), and blocks the add-to-cart action until a selection is confirmed.
- **Option B:** The system shows a confirmation message in the modal that clearly flags the attributes as defaults (e.g., _"Adding with default size: S and color: White — is this correct?"_) and provides an easy way to change selection.

---

## Actual Result

The product is immediately added to the cart using **Size: S** and **Color: White** (system defaults). The confirmation modal appears with the message:  
> _"Product successfully added to your shopping cart"_

The modal shows the attribute values but does not indicate they are defaults or prompt the user to verify. There is no blocking mechanism or warning before the add-to-cart action completes.

---

## Impact

- **Business Impact:** Users may complete orders with incorrect size/color combinations, leading to returns, complaints, and customer dissatisfaction.
- **UX Impact:** Violates the principle of least surprise — the system acts without clear user intent.
- **Reproducibility:** 100% reproducible on first visit or when attributes are not explicitly touched.

## Suggested Fix

Implement an explicit attribute selection guard before the add-to-cart action:

1. Track whether the user has **actively interacted** with each required attribute selector (size, color)
2. If any required attribute has not been explicitly chosen by the user, display an inline warning and prevent cart addition:
   > _"Please select a Size and Color before adding this item to your cart."_
3. Alternatively, display a confirmation step in the modal that highlights the selected attributes and includes a clear "Change" link to return to the PDP for reselection.
