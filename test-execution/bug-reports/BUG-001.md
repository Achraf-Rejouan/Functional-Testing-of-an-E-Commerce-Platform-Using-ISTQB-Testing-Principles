## Description

When a user clicks the search icon (or presses Enter) with an **empty search bar**, the application does not block the request with a client-side validation message. Instead, it silently redirects the user to the search results page, which displays the generic message _"No matches were found for your search."_

This behavior is inconsistent with standard UX practices for search inputs, where an empty query should either:
1. Be ignored entirely (no navigation occurs), or
2. Display an inline validation message prompting the user to enter a keyword.

Processing and routing an empty search query to the server wastes a network round-trip and provides no useful feedback to the user.

---

## Steps to Reproduce

1. Navigate to `https://demo.prestashop.com/#/en/front`
2. Click on the **search bar** (magnifying glass icon) in the top navigation
3. Do **not** type anything — leave the search field completely empty
4. Press **Enter** or click the search submit button

**Expected frequency:** Reproducible 100% of the time.

---

## Expected Result

The search submission is blocked before navigating away. A validation message is displayed, such as:  
> _"Please enter a search term."_  

No page navigation occurs and the user remains on the current page with focus returned to the search bar.

---

## Actual Result

The browser navigates to the search results page (`?controller=search&s=`) and displays:  
> _"No matches were found for your search"_  
> _"Please re-enter the search term in the search bar."_

No client-side validation or blocking occurs before the redirect.

## Suggested Fix

Add client-side validation in the search form submission handler to check for empty or whitespace-only input before triggering navigation:

```javascript
// Pseudocode suggestion
searchForm.addEventListener('submit', (e) => {
  if (searchInput.value.trim() === '') {
    e.preventDefault();
    showValidationMessage('Please enter a search term.');
    searchInput.focus();
  }
});
```
